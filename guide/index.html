Welcome to Mayhem

What is Mayhem?

Mayhem is an all-in-one framework for developing large applications in TypeScript or JavaScript, following these key tenants:

  • Predictability: As much as possible, the framework must be designed to minimise surprise. Mayhem uses as little magic as possible in order to accomplish its goals, while still being exceptionally powerful & easy to use, with a minimum of rote.
  • Extensibility: It should be possible for all parts of the framework to be easily extended or replaced with newer, better components. This allows many different classes of applications to be written, including those not envisioned when the framework was originally written.
  • Correctness: When writing applications on a deadline, it’s frustrating to run into defects in tools. Mayhem was architected from the ground up to be extremely modular, testable, and easy to understand in order to reduce the maintenance costs of using Mayhem for your own apps.

While you will see some performance optimisations within Mayhem that are designed to ensure it is as fast as possible, Mayhem is not built for speed. We will not compromise clarity and maintainability for the sake of performance. If a particular design is faster but extremely difficult to troubleshoot, we will choose the simpler, clearer design. We hope that you will see the benefit of this decision in increased productivity and less time spent troubleshooting your Mayhem apps!

Installation

Depending upon the platform you are planning on developing against with Mayhem, there are three different ways in which installation can be performed:

  • For Node.js: Mayhem can be installed on Node.js by running npm install mayhem. This will install a copy of Mayhem that has been pre-compiled for Node.js using the CommonJS module format, plus TypeScript ambient declarations that can be used when writing a Mayhem application for Node.js.
  • For browsers & mobile devices: Mayhem can be installed for browsers & mobile devices by running bower install mayhem. This will install a copy of Mayhem that has been pre-compiled for asynchronous environments using the AMD module format, plus TypeScript ambient declarations that can be used when writing a Mayhem application for browsers or mobile devices.
  • For developing Mayhem: The original TypeScript source for Mayhem can be installed from our GitHub repository by running git clone --recursive git@github.com:SitePen/mayhem.git. The README contains additional instructions on completing installation of the development version of Mayhem.

Because Mayhem is written in TypeScript, future versions of Mayhem can be compiled to the standard EcmaScript 6 module format once it has been finalised and is available within the TypeScript compiler.
